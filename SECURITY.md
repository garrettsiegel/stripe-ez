# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 1.x | Yes |
| < 1.0.0 | No |

## Reporting a Vulnerability

Please report suspected vulnerabilities privately.

Preferred channels:

- GitHub Security Advisory: [Create a private advisory](https://github.com/garrettsiegel/stripe-ez/security/advisories/new)
- If advisories are unavailable for your account, open a private maintainer contact request via [GitHub Discussions](https://github.com/garrettsiegel/stripe-ez/discussions)

Please include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested remediation (if known)

## Response Expectations

- Initial acknowledgement: within 72 hours
- Triage and severity assessment: within 7 days
- Status updates: at least every 7 days until resolution

We ask that you do not publicly disclose the issue until we have had a reasonable opportunity to investigate and publish a fix.

## Scope

This policy covers:

- The `stripe-ez` CLI package
- This repository and its official release artifacts

Out of scope:

- Third-party services and dependencies not maintained in this repository
